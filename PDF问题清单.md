## 五、数据一致性

### ~~13. targetMD.md 与 PDF 输出内容不一致~~ ✅ 已修复
- **修复内容**：重写 `processList` 中嵌套列表的父项文本提取逻辑。原代码只取 `text` 类型 token，改进为收集所有非 `list` 类型 token（包括 `paragraph` 等），避免潜在的内容截断。

### 14. `**bold**` 标记字面量出现在 PDF 文本中 ✅ 已修复
- **问题描述**：DS-portal 招聘系统后、以及"线上问题排查"结果的 `**结果**` 在 PDF 中以字面量 `**` 出现。
- **根因分析**：两个独立 bug：
  1. `bullet` 方法中的内联标签正则 `/^\*\*(.+?)\*\*：/` 使用 lazy `(.+?)`，当文本中包含多对 `**` 时（如 `**DS-portal...**（...）...**结果**：`），lazy 匹配会跨越中间的 `**` 边界，导致 label 捕获包含 `**` 字面量。
  2. `bulletInlineLabel` 方法使用 `splitTextToSize` 对含 `**` 的原始内容字符串换行，可能在 `**` 标记中间断开（如 `**结` / `果**`），导致 `parseSegments` 无法匹配不完整的 `**` 对。
- **修复内容**：
  1. 将内联标签正则改为 `/^\*\*([^*]+)\*\*：/`，使用 `[^*]+` 禁止捕获组内出现 `*`，确保只匹配最近的 `**...**：` 模式。
  2. 重写 `bulletInlineLabel` 的内容渲染逻辑，改用与 `wrappedRichTextBold` 相同的 segment-aware 换行方式：先对 plain text（去除 `**`）做 `splitTextToSize`，再将换行结果映射回 segment 进行渲染，从根本上避免 `**` 被断开。同时为每个内容行添加 `this.need()` 分页检查，修复长内容跨页丢失的问题。

### 15. 长文本内容跨页截断（DS-portal 描述末尾丢失） ✅ 已修复
- **问题描述**：DS-portal 招聘系统描述在页面底部被截断，后续内容（"终发证的全流程…"）未出现在下一页。
- **根因分析**：旧版 `bulletInlineLabel` 的 `_renderInlineBold` 方法缺少 `this.need()` 分页检查，当内容跨越页面边界时，超出部分被写入页面外导致丢失。
- **修复内容**：新实现为每个内容渲染行都调用 `this.need(LH_BODY)`，确保跨页时正确换页。

### ~~16. h2 分隔线未显示~~ ✅ 已修复
- **修复内容**：`h2` 方法新增 `doc.line()` 绘制分隔线，使用 `GAP_H2_RULE = 6.0pt` 和 `GAP_H2_CONTENT = 12.0pt` 常量控制间距。分隔线为 `C_ACCENT` 颜色、`1.2pt` 线宽，贯穿内容区域宽度。同时修复了 `this.y` 定位，确保分隔线与内容不重叠。

### ~~17. 个人信息表格缺少边框线~~ ✅ 已修复
- **修复内容**：`personalInfo` 方法新增 `rect()` 绘制外边框和表头背景色，`line()` 绘制垂直/水平分隔线。表头使用 `(232, 236, 240)` 灰色背景填充。同时修复个人网站与表格的间距（增至 18pt）。

### 22. 小黑圆点（bullet）垂直位置偏移 🔴 高
- **现象**：列表项前的实心小黑圆点位置偏低，出现在文字基线附近，而不是与文字垂直居中对齐。
- **根因分析**：jsPDF `text()` 的 y 坐标是文字基线位置，而 CJK 字符的视觉中心在基线上方约 `fontSize * 0.4` 处。原代码使用 `dotCenterY = this.y + 2.2`（基线下方 2.2pt），导致圆点严重偏下。
- **修复内容**：将圆点中心改为 `dotCenterY = this.y - 3.5`，使圆点与 CJK 字符视觉中心对齐。同时移除了调试用的 `console.log` 语句。

### ~~18. 整体行间距偏大，排版松散~~ ✅ 已修复
- **修复内容**：`LH_BODY` 从 `17.5pt` 减小至 `15.5pt`，使行间距更紧凑，与参考PDF一致。

### ~~19. 嵌套列表缩进层次不够明显~~ ✅ 已修复
- **修复内容**：`NESTED_INDENT` 从 `16.0pt` 增大至 `20.0pt`，使嵌套列表的层级缩进更明显。

### ~~20. 四列布局方案/结果列过窄，文字过度换行~~ ✅ 已修复
- **修复内容**：`bulletFourColFull` 中方案/结果的列宽比例从各 `30%` 增大至各 `38%`，减少文字过度换行。

### ~~21. 个人优势/专业技能中单词被错误换行~~ ✅ 已修复
- **修复内容**：重写 `safeWrapText` 函数，将文本先分词（英文单词为单个 token，CJK 字符各为独立 token），再按 token 贪心填充每行，确保英文单词不会被从中间断开。
